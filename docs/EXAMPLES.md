# 使用案例和示例

## 📚 案例1: React开发工程师求职

### 场景描述
李明是一名有3年工作经验的React开发工程师，想在北京寻找年薪25K-35K的全职工作。

### 配置步骤

1. **打开插件**
   - 点击Chrome右上角的插件图标
   - 插件弹出窗口

2. **配置筛选条件**
   ```
   岗位名称: React 开发
   招聘网站: BOSS直聘
   城市: 北京
   区域: 朝阳区 (可选)
   年龄范围: 25-35岁
   薪资范围: 25K-35K/月
   岗位类型: 全职
   技能标签: 
     - React
     - TypeScript
     - Node.js
     - Vue.js
   ```

3. **采集数据**
   - 点击"点击刷新采集数据"按钮
   - 等待3-5秒采集完成

4. **查看结果**
   - 自动切换到职位列表页面
   - 查看找到的职位总数
   - 浏览匹配度最高的职位

### 预期结果
- 找到15个职位
- 其中8个职位匹配度>85%
- 可以快速筛选出最适合的职位

## 📚 案例3: 全栈工程师多城市搜索

### 场景描述
张先生是全栈工程师，可以考虑北京、上海和杭州三个城市，想找到年薪30K-50K的工作。

### 工作流程

```
第一轮搜索 - 北京
├─ 岗位名称: 全栈开发
├─ 城市: 北京
├─ 薪资: 30K-50K
└─ 采集 → 找到8个职位

第二轮搜索 - 上海
├─ 修改城市: 上海
├─ 其他配置不变
└─ 采集 → 找到12个职位

第三轮搜索 - 杭州
├─ 修改城市: 杭州
├─ 其他配置不变
└─ 采集 → 找到6个职位

综合分析
├─ 比较三个城市的机会数量
├─ 分析各地薪资水平
└─ 决定投递策略
```

### 技能标签建议
```
必需:
- React/Vue (前端框架)
- Node.js (后端框架)
- MySQL (数据库)

加分:
- TypeScript
- Docker
- Git
- Webpack
```

## 🎓 最佳实践总结

### 搜索技巧

1. **使用具体的岗位名称** 而非泛泛而谈
   - ✅ "React 前端开发工程师"
   - ❌ "开发"

2. **合理设置薪资范围**
   - 不要过于保守 (失去好机会)
   - 不要过于激进 (浪费搜索结果)
   - 推荐幅度: 10K左右

3. **定期更新技能标签**
   - 每月评估新掌握的技能
   - 删除已不使用的技能
   - 关注行业热点技能

4. **多维度分析结果**
   - 职位数量反映需求
   - 薪资范围反映市场价值
   - 所需技能反映学习方向

### 采集频率建议

- **求职阶段**: 每周1-2次
- **在职学习**: 每月1-2次
- **长期规划**: 每季度1次

### 数据分析

收集多次搜索结果后，可以分析：
- 平均薪资趋势
- 技能需求变化
- 职位机会分布
- 竞争强度变化
